## Feature 07: IndexedDB Cache Migration

### Important

**[I-1] `getDB()` race condition khi nhiều caller gọi đồng thời lần đầu**
- File: `lib/cache-db.ts`
- Nếu 2 caller gọi `getDB()` cùng lúc trước khi DB mở xong, cả 2 đều thấy `db === null` → mở 2 connection → connection đầu bị leak (không close, không track)
- Fix: Dùng shared `openingPromise` variable:
  ```ts
  let openingPromise: Promise<IDBDatabase> | null = null;
  function getDB(): Promise<IDBDatabase> {
    if (db) return Promise.resolve(db);
    if (openingPromise) return openingPromise;
    openingPromise = new Promise(...);
    return openingPromise;
  }
  ```

### Minor

**[M-1] Thiếu `onversionchange` handler — block DB upgrade trong tương lai**
- File: `lib/cache-db.ts`
- Nếu tăng `DB_VERSION` sau này, connection cũ không tự close → upgrade bị block vô hạn
- Fix: Thêm `db.onversionchange = () => { db.close(); db = null; };` trong `onsuccess`

**[M-2] `getCacheSize` load toàn bộ data vào memory để đếm bytes**
- File: `lib/cache-manager.ts`
- `dbGetAll()` fetch ALL topics kể cả `posts` array → với cache 50MB sẽ gây memory spike
- Fix: Dùng `StorageManager.estimate()` hoặc lưu `byteSize` field trên mỗi topic

**[M-3] `SAVE_CACHED_TOPIC` read-modify-write không atomic**
- File: `entrypoints/background/index.ts`
- 2 concurrent save messages đọc cùng `existing`, merge riêng → save cuối cùng ghi đè save trước
- Xác suất thấp nhưng có thể xảy ra nếu user trigger summary + opinions cùng lúc

## Feature 08: Topic URL, Auto-update, CacheIndicator

### Important

**[I-1] `autoUpdateCachedTopic` có thể ghi đè title hợp lệ bằng empty string**
- File: `entrypoints/sidepanel/App.vue`, dòng 74-76
- Nếu content script parse title thất bại → `detect.title = ''` → `hasChanges = true` (vì `cached.title !== ''`) → ghi đè title cached bằng empty string
- Fix: Thêm guard `if (!detect.title) return;` hoặc `cached.title !== detect.title && detect.title`

**[I-2] `CacheIndicator.currentPosts` prop ngữ nghĩa mập mờ**
- File: `CacheIndicator.vue`
- `currentPosts` nhận từ `livePostCount` — khi tab không khớp, fallback = `topic.totalPosts` = `cachedPosts` → `newPostCount = 0` → ẩn button. Đúng hành vi nhưng semantic confusing
- Fix: Truyền `null` khi không có live data, guard trong component

### Minor

**[M-1] `SummaryView.navigateToTopic` là dead code**
- File: `SummaryView.vue`
- Navigation đã delegate sang `TopicMeta` (prop `url`) → function `navigateToTopic` trong SummaryView không còn được gọi
- Fix: Xóa function

**[M-2] Hai implementation normalize URL khác nhau**
- `cache-manager.normalizeUrl` (đảm bảo trailing `/`) vs `SummaryView.isSameTopicUrl` (không xử lý trailing `/`)
- Có thể gây mismatch trong edge case. Nên dùng chung 1 implementation

## Feature 09: Detect Thread, Search/Sort, Opinion Bars, OpinionsView

### Important

**[I-1] `isThreadPage` XF1: selector `#messageList .message` gây false positive**
- File: `entrypoints/content/index.ts`, dòng 16-17
- `#messageList .message` match cả trang "Recent Posts", member profile posts — không phải thread page
- Fix: Bỏ branch thứ 2, chỉ giữ `li.message .messageText` (đủ cho thread pages)

**[I-2] Regex parse supporter count quá strict với ký tự cuối**
- File: `SummaryContent.vue`, dòng 41
- Regex `\)\s*$` yêu cầu `)` là ký tự cuối. LLM có thể output `(N người ủng hộ):` hoặc `(N người ủng hộ).` → regex fail → không render bar
- Fix: Cho phép trailing punctuation: `\)[.,:]?\s*$`

**[I-3] `isThreadPage` DOM query có thể fail trên trang load chậm**
- File: `entrypoints/content/index.ts`
- `DETECT_XF` có thể đến trước khi DOM render xong → `article.message` chưa có → trả `undefined`
- Trước Feature 09 trả data dù sai → không ảnh hưởng. Giờ trả `undefined` = không detect → user thấy empty
- Mitigation: Vấn đề pre-existing, nhưng giờ hệ quả rõ hơn. Có thể thêm retry logic ở App.vue

### Minor

**[M-1] OpinionsView: response bất đồng bộ có thể ghi đè topic sai khi switch nhanh**
- File: `OpinionsView.vue`
- `loadedTopicUrl` set ngay lập tức, nhưng `GET_CACHED_TOPIC` async có thể resolve sau khi đã switch sang topic khác → data cũ ghi đè data mới
- Fix: Capture `topic.url` vào local variable, check `loadedTopicUrl.value === url` sau `await`

## Feature 10: Smart Scraping

### Critical

**[C-1] Incremental scrape bắt đầu từ `cachedPages` thay vì `cachedPages + 1` — luôn re-fetch trang cuối đã cache**
- File: `SummaryView.vue`, dòng 319-322
- `startPage = Math.max(1, cachedPages)` → trang cuối đã scrape bị fetch lại mỗi lần incremental
- Posts dedup đúng nên không ảnh hưởng kết quả, nhưng lãng phí bandwidth + thời gian
- Fix: `startPage = Math.max(1, cachedPages + 1)` (khi cachedPages > 1)

### Important

**[I-1] Article extraction không có timeout — fetch bài báo có thể hang vô hạn**
- File: `lib/scrapers/article-extractor.ts`, dòng 15
- `fetch(url)` không có `AbortController` timeout → trang báo chậm block background SW
- Fix: Thêm `AbortController` với timeout 10-15 giây

**[I-2] Fallback "largest text block" trong article-extractor chọn sai element**
- File: `lib/scrapers/article-extractor.ts`, dòng 56-65
- Chọn `div/section` có `textContent` dài nhất → thường là outermost container (= toàn bộ trang)
- Truncate 3000 chars → lấy phần đầu trang (thường là nav/sidebar)
- Fix: Ưu tiên elements có text density cao (text length / child count), hoặc dùng heuristic `<article>`, `<main>`, `.post-body`, `.entry-content`

**[I-3] Article posts (postNumber âm) bị loại khỏi incremental summarize**
- File: `SummaryView.vue`, dòng 412-414
- `newPosts = posts.filter(p => p.postNumber > lastPostNumber)` → postNumber âm luôn < lastPostNumber → bị filter
- **Hậu quả:** Context bài báo gốc chỉ có tác dụng ở lần tóm tắt đầu, mất khi incremental update
- Fix: Tách riêng article posts và forum posts khi filter: `posts.filter(p => p.postNumber < 0 || p.postNumber > lastPostNumber)`

**[I-4] Sparse segment array khi user tóm tắt segment không tuần tự**
- File: `SummaryView.vue`, dòng 503-505
- Tóm tắt segment 0 rồi skip sang segment 3 → array có holes `[seg0, undefined, undefined, seg3]`
- Persist vào IndexedDB → reload lại có `undefined` entries
- Fix: Dùng `Map<number, TopicSegment>` hoặc fill missing indices với empty objects

### Minor

**[M-1] `scrapePageRange()` trả `totalPages: endPage` thay vì tổng số trang thực tế**
- File: `lib/scrapers/page-loader.ts`
- Không ảnh hưởng hiện tại vì caller không dùng field này, nhưng misleading

**[M-2] Message "Đã tải N bài báo gốc" hiện dưới heading "Một số trang bị bỏ qua:"**
- File: `SummaryView.vue`, dòng 382
- Success message push vào `scrapingWarnings` → hiện dưới heading warning → confusing UX
- Fix: Dùng array riêng `scrapingInfo` cho informational messages
